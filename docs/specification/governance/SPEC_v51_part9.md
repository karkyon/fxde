## 1. 本ツールの本質的価値（再定義）

### 1.1 AI 分析ツール vs 完全自動売買

| 軸 | 完全自動売買 | **AI 分析ツール（本ツール）** |
|----|------------|--------------------------|
| AI の役割 | 全判断 + 自動発注 | アシスタント（人間が最終判断）|
| 責任所在 | AI | トレーダー本人 |
| 法的リスク | 高（投資助言業に抵触可能性）| 低（情報提供のみ）|
| 個人開発難易度 | ★★★★★ | ★★★☆☆ |
| ビジネス化しやすさ | 低 | **高** |
| 失敗時の損失 | 自動で拡大 | 人間が止められる |
| 相場急変への対応 | モデル崩壊リスク | 人間が気づいて回避 |

**結論: 本ツールは「AI 分析ツール」として設計する。**  
AI は優秀なアシスタントであり、最終決断はトレーダーが行う。

### 1.2 AI が担う役割の定義（確定）

```
AI の仕事（本ツール）:
  ① 市場状況の整理・可視化（スコアエンジン）
  ② エントリー候補の提示（チャートパターン + MTF）
  ③ 未来シナリオの確率的提示（予測チャート）
  ④ 精度の自己検証・改善（フィードバックループ）
  ⑤ 市場状況を平易な日本語で解説（AI 要約）

人間の仕事（トレーダー）:
  ① 最終エントリー判断
  ② ポジション管理（SL/TP の移動）
  ③ 想定外のニュース・イベントへの対応
  ④ ルールのカスタマイズ（プリセット選択）
```

### 1.3 本ツールが解決する本質的問題

```
一般的なトレードツールの価値提案:
  「より良い予測をする」

本ツールの本質的価値:
  「感情的な判断を排除し、統計的に有利な局面だけでエントリーする仕組み」

勝てるトレーダーとの違いは「分析力」ではなく「規律」。
本ツールはその規律を AI と設計で強制する。

ターゲットユーザーが抱える本当の問題:
  「分析はできる。でも FOMO やリベンジトレードで勝ちを消してしまう」
  ↓
  この問題に直接刺さるのが本ツールの強み。
```

## 2. 競合分析と差別化戦略

### 2.1 競合マップ

| ツール | 強み | 弱み | 月額 |
|--------|------|------|------|
| TradingView | チャート最強・コミュニティ豊富 | 予測なし・日本語対応弱い | $15〜$60 |
| TrendSpider | AI パターン検出 | 日本語なし・FX やや弱い | $33〜$97 |
| Trade Ideas | 株 AI スキャナー | FX 非対応・英語のみ | $118〜$228 |
| MetaTrader EA | 自動売買可能 | 専門知識必要・管理大変 | 無料〜 |
| **本ツール** | **下記 5 点** | 認知度ゼロ（初期）| ¥0〜¥4,980 |

### 2.2 本ツールの 5 つの差別化ポイント

```
① 日本語 + 初心者モード
   → 競合は全て英語中心。
     日本の FX 個人投資家（世界最大の外為取引量を持つ市場）向けに特化。
     専門用語に ( ) で説明を付ける設計は競合にない。

② 規律エンジン（スコアロック）
   → 感情的エントリーを物理的に防ぐ機能は競合にない。
     「負けにくくする」という訴求は初心者に強く刺さる。
     連敗後の強制冷却タイマーは行動経済学に基づく設計。

③ 予測精度の自己検証・公開（v6 以降）
   → 将来的に「このAIの予測精度を継続検証し、結果を公開予定」という信頼性訴求を行う。
     競合は精度を公開しない（できない）。
     ウォークフォワード検証による過剰最適化の排除が差別化になる。
     v5.1 では精度検証 UI のみ提供し、Prediction Service はスタブ実装のため実測値は v6 以降で取得する。

④ MTF × 類似局面検索の組み合わせ（v6 本実装予定）
   → 形状ベースの類似チャート検索は個人向けツールに希少。
     「過去47件の類似局面では63%が上昇しました」という統計的根拠が
     ユーザーの信頼を獲得しやすい。
     v5.1 では Prediction Service をスタブ実装とし、v6 で DTW を本実装する。

⑤ 心理分析・規律可視化
   → 行動経済学 × トレードという切り口は差別化になる。
     「連敗後の勝率崩壊グラフ」「スコア帯別期待値」など、
     自分のデータで「ルールを守る価値」を証明できる。
```

### 3.3 DB ロール管理（v5 運用方法）

```typescript
// v5 では ADMIN が手動でロールを変更する
// apps/api/src/admin/admin.service.ts

async setUserRole(targetUserId: string, role: UserRole): Promise<void> {
  await this.prisma.user.update({
    where:  { id: targetUserId },
    data:   { role },
  });
  await this.auditLog.write({
    action:     'user.role_change',
    targetType: 'User',
    targetId:   targetUserId,
    metadata:   { newRole: role },
  });
}

// PATCH /api/v1/admin/users/:id/role — ADMIN のみ
```

## 6. 法的・倫理的考慮事項

### 6.1 投資助言業との境界線

```
投資助言業（金融商品取引法 2 条 8 項 11 号）に該当するもの:
  × 「今すぐ EUR/USD を買いなさい」
  × 「この通貨ペアは必ず上がります」
  × 特定の有価証券に対する個別具体的な売買推奨

情報提供として適法なもの:
  ✅ 「過去 47 件の類似局面では 63% が上昇しました」（統計情報）
  ✅ 「スコア 72 点（待機目安: 75 点）」（計算結果）
  ✅ 「AI が分析した参考情報です。判断はご自身で」（免責付き情報）
  ✅ テクニカル指標の状態表示（MA クロス等）
```

### 6.2 必須免責事項（全ページ・常時表示）

```
※ 本ツールは情報提供のみを目的とし、投資助言ではありません。
  FX 取引には元本割れのリスクがあります。
  投資の最終判断は必ずご自身の責任で行ってください。
  過去の予測精度は将来の結果を保証するものではありません。
```

### 6.3 データプライバシー

```
保存しないデータ:
  - 証券口座のログイン情報
  - 実際のポジション（API で取得しない）
  - クレジットカード番号（Stripe が直接管理）

保存するデータ（ユーザーが手動入力したもの）:
  - トレード記録（symbol, price, pnl 等）
  - 設定値
  - 振り返りコメント

GDPR / 個人情報保護法対応:
  - ユーザー退会時に全データを削除（status=SUSPENDED → 90 日後に物理削除）
  - データエクスポート機能（CSV）を提供
  - プライバシーポリシーページを設置
```

## 11. 設計上の注意事項（変えてはいけないこと）

```
1. スコア閾値は「下げない」のがルール
   → ユーザーが「少し下げれば入れたのに」と思っても、
     それがシステムの価値。妥協しない設計のままにする。

2. EntryDecision の優先順は変えない
   forceLock > cooldown > eventLock > riskNg > scoreLow
   → この順番は行動経済学的根拠がある。変更は全テスト通過後のみ。

3. パターンボーナスは「最高点 1 つのみ」
   → 重複加算を許可しない。複数パターン検出は「1 番強いもの」のみ採用。

4. RR < 1.0 は絶対にエントリーさせない
   → scoreThreshold を 0 に設定しても RR_NG は回避できない。

5. RefreshToken は HttpOnly Cookie のみ
   → localStorage への AT 保存も禁止。
     セキュリティ要件は妥協なし。

6. 外部 API の障害時はキャッシュで動き続ける
   → 「API が落ちたからスコアが出ない」は許さない。
     TTL 内のキャッシュで degraded mode 動作を保証する。

7. v5.1 の Prediction Service はスタブ実装のみ
   → DTW / HMM / 類似局面検索は v6 設計資料扱い。
     v5.1 では STUB_PREDICTION_RESULT 固定 JSON を返却する実装のみとする。
```
